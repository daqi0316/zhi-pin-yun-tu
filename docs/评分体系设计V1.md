# 简历评分 & 面试评分体系 V1.0

> 依据：学术界 KSAO 框架 + 结构化面试 BARS 方法论 + 国内大厂实战经验
> 应用场景：用于智聘云图招聘系统的 AI 人才评估

---

## 一、简历评分（Resume Scoring）

### 评分模型

```
综合得分 = 技能匹配 × 30% + 经验匹配 × 25% + 教育背景 × 10%
         + 能力信号 × 15% + 稳定性 × 10% + 薪酬匹配 × 10%
```

> 权重依据：参考 Kristof-Brown et al. (2005) 对 P-J Fit 和 P-O Fit 的元分析结论，
> 结合国内大厂（腾讯/阿里/字节）招聘时对技能和经验的实际重视程度调整。

### 维度详解

#### 维度一：技能匹配（权重 30%）

| 等级 | 分值 | 定义 | 示例 |
|------|------|------|------|
| S | 90-100 | 候选人技能覆盖 JD 全部必须技能 + 部分加分技能 | JD 要 Java/Spring/Redis，候选人还会 Kafka、MySQL |
| A | 75-89 | 覆盖全部必须技能 | JD 要 Java/Spring/Redis，候选人全有 |
| B | 60-74 | 覆盖核心必须技能，缺少 1-2 项 | JD 要 Java/Spring/Redis/MySQL，候选人缺 MySQL |
| C | 40-59 | 覆盖部分必须技能，缺少 2 项以上 | JD 要 5 项技能，候选人只有 2-3 项 |
| D | 0-39 | 几乎不匹配 | 技能栈完全不相关 |

**计算公式：**
```
技能匹配分 = (匹配必须技能数 / 必须技能总数 × 100) × 0.7
           + (匹配加分技能数 / 加分技能总数 × 100) × 0.3
```

**数据来源：** 候选人简历 parsed skills vs JD 录入的技能要求

#### 维度二：经验匹配（权重 25%）

| 等级 | 分值 | 定义 |
|------|------|------|
| S | 90-100 | JD 要求 3-5 年，候选人 4-6 年，且行业高度相关 |
| A | 75-89 | 年限在合理区间内，行业相关 |
| B | 60-74 | 年限接近但超出或不足 1-2 年 |
| C | 40-59 | 年限偏差较大（±3 年以上） |
| D | 0-39 | 年限完全不匹配或无相关经验 |

**计算公式：**
```
年限匹配度 = 1 - |候选人年限 - 目标年限| / 目标年限
经验匹配分 = 年限匹配度 × 60 + 行业相关性 × 40
```

> 行业相关性：同行业(100) / 相关行业(70) / 跨行业(30) / 无相关(0)

#### 维度三：教育背景（权重 10%）

| 等级 | 分值 | 定义 |
|------|------|------|
| S | 90-100 | 博士 / 985 硕士，专业完全对口 |
| A | 75-89 | 硕士 / 985 本科，专业相关 |
| B | 60-74 | 本科，专业相关或硕士跨专业 |
| C | 40-59 | 本科，专业不相关 |
| D | 0-39 | 大专及以下 |

#### 维度四：能力信号（权重 15%）

| 信号类型 | 分值权重 | 说明 |
|---------|---------|------|
| 职级晋升 | 30% | 近 5 年是否有晋升记录（专员→经理→总监） |
| 项目复杂度 | 30% | 是否有千万级用户 / 高并发 / 核心项目经验 |
| 大厂背景 | 20% | 是否有头部公司（BAT/字节/TMD/外企）经历 |
| 管理经验 | 20% | 是否有团队管理或带人经验 |

**学术依据：**
- 职级晋升速度反映学习能力和成长潜力（Seibert et al., 2013）
- 大厂背景可作为能力信号的替代指标（Bidwell, 2011）
- 管理经验与领导力正相关（DeRue et al., 2010）

#### 维度五：稳定性（权重 10%）

| 等级 | 分值 | 定义 |
|------|------|------|
| S | 90-100 | 每份工作 3-5 年，无短跳 |
| A | 75-89 | 平均 2-3 年，合理范围 |
| B | 60-74 | 有 1-2 次短跳（<1 年）但有合理解释 |
| C | 40-59 | 多段工作 <1 年 |
| D | 0-39 | 频繁跳槽，无明显职业积累 |

**计算公式：**
```
平均任职时长 = 候选人总工作年资 / 公司数
稳定性分 = min(平均任职时长 / 3, 1) × 100
```
> 3 年为理想基准线，超过 3 年按满分计

#### 维度六：薪酬匹配（权重 10%）

| 等级 | 分值 | 定义 |
|------|------|------|
| S | 90-100 | 期望薪资在预算范围内 ±10% |
| A | 75-89 | 偏差在 10%-20% 之间 |
| B | 60-74 | 偏差在 20%-30% 之间 |
| C | 40-59 | 偏差在 30%-50% 之间 |
| D | 0-39 | 偏差超过 50% |

## 二、面试评分（Interview Scoring）

### 评分模型

```
面试得分 = 专业技能 × 30% + 问题解决 × 20% + 沟通表达 × 20%
         + 团队协作 × 15% + 文化匹配 × 15%
```

> 依据：美国 OPM 结构化面试标准 + 国内大厂面试评分卡（北森胜任力词库 GENE3）

### 评分方式

每项采用 **BARS（行为锚定评分法）**，1-5 分制，每分对应具体行为描述。
最终按权重汇总为百分制。

#### 维度一：专业技能（权重 30%）

| 分数 | 行为锚定描述 |
|------|-------------|
| 5 | 对核心技能有深度理解，能回答原理性问题，有实际项目落地经验 |
| 4 | 熟练掌握核心技能，能独立完成复杂任务 |
| 3 | 具备基本技能，能完成常规任务，但深度不足 |
| 2 | 技能基础薄弱，仅停留在理论层面 |
| 1 | 几乎不具备岗位要求的核心技能 |

> 面试官需结合 STAR 追问，至少要求候选人提供一个具体项目案例

#### 维度二：问题解决（权重 20%）

| 分数 | 行为锚定描述 |
|------|-------------|
| 5 | 能系统性地拆解复杂问题，提出多套方案并权衡优劣 |
| 4 | 能快速定位问题核心，给出合理的解决路径 |
| 3 | 能解决常规问题，但面对复杂问题缺乏系统性 |
| 2 | 只能按照既定流程解决问题，缺乏独立判断 |
| 1 | 面对问题无从下手，缺乏逻辑思维 |

#### 维度三：沟通表达（权重 20%）

| 分数 | 行为锚定描述 |
|------|-------------|
| 5 | 表达清晰有条理，能根据听众调整表达方式，善于说服和推动 |
| 4 | 表达流畅，逻辑清晰，能有效传递信息 |
| 3 | 表达基本清楚，偶尔需要追问才能理解要点 |
| 2 | 表达混乱，抓不住重点 |
| 1 | 沟通困难，难以理解其意图 |

#### 维度四：团队协作（权重 15%）

| 分数 | 行为锚定描述 |
|------|-------------|
| 5 | 善于跨团队协作，能主动补位，推动团队目标达成 |
| 4 | 良好合作者，能配合团队完成任务 |
| 3 | 能完成自己分内工作，但较少主动协作 |
| 2 | 合作意识薄弱，倾向于单打独斗 |
| 1 | 无法融入团队，有协作冲突历史 |

#### 维度五：文化匹配（权重 15%）

| 分数 | 行为锚定描述 |
|------|-------------|
| 5 | 价值观高度契合，对行业有热情，职业规划与公司方向一致 |
| 4 | 价值观基本一致，求职动机合理 |
| 3 | 无明显冲突，但匹配度一般 |
| 2 | 存在一定价值观差异或动机不纯 |
| 1 | 明显不匹配（如只看重薪资、对公司业务缺乏认同） |

## 三、综合总分

```
综合得分 = 简历评分 × 0.4 + 面试评分 × 0.6
```

> 权重逻辑：面试的预测效度（0.5-0.6）高于简历筛选（0.2-0.3），
> 依据 Schmidt & Hunter (1998) 人事选拔方法效度元分析

### 等级映射

| 等级 | 综合得分 | 建议 |
|------|---------|------|
| S | 90+ | 强烈推荐，直接推进 |
| A | 75-89 | 推荐，正常推进 |
| B | 60-74 | 备选，对比其他候选人后再定 |
| C | 40-59 | 暂缓，除非有特殊优势 |
| D | <40 | 淘汰 |

## 四、后续待完善

1. **评分权重的岗位定制化** — 不同岗位（技术/产品/运营）的维度权重应有差异
2. **面试官评分校准** — 多位面试官打分时的偏差校准机制
3. **AI 辅助评分** — 接入 LLM 做简历与 JD 的语义匹配，输出初步评分后再由人工校验
4. **评分历史追踪** — 记录每次评分，形成候选人成长轨迹
